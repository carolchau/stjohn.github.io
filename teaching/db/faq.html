  2. What is the difference between a SUBCLASS and a WEEK ENTITY is both of them occur when one entity depends on any attribute of another entity?

    Both of these depend on other E.S.'s., but they are different.

    In E/R diagrams, a weak entity set is an E.S. that depends on another E.S. set for its key (example: logins and keys). To have a weak entity set, you must have a relationship between it and another E.S. The supporting relationship and E.S. provide the key for the weak E.S.

    A subclass is a collection of entities from a given E.S. that may have extra properties the original E.S. does not have (example: murder-mysteries are a subclass of movies, which have an extra attribute weapon).

  3. I wanted to ask in general, but could we have telnetted into comet and used the psql application to help us solve the SQL problems (or check) in the book ?

If you have a machine running X Windows (usually unix based or a package like ReflectionX installed if you're using MS Windows), you can even use pgaccess. In general, when you telnet in, it will look just like the terminal window you have in the lab. You won't be able to use any graphics programs (like pgaccess or some of the editors), but all the text-based commands will work fine.

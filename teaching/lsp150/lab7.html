Communication

Today's lab focuses on communication. The first set of programs beams a number to another robot, who responds by beeping. If you have your robot with a bumper and light sensor, you should also try the RoboTag.nqc program.

Design requirements: This lab assumes that the two motors are wired to output ports A and C. Further, each motor is attached to a means of locomotion (legs, treads, feet, etc.). When using the light sensor, it is wired to sensor port 2. The touch sensor is wired to sensor port 1 and is attached to a bumper).

Sending Messages

Pick a number between 1 and 7.  Write a program called sendMessage.nqc that will send that number.  To test your program, ownload the beep.nqc program to one robot and your send message program to another robot.  So, one robot will be running the beep.nqc program and the other will be running sendMessage.nqc.

Next, write a program that will pick a random number between 1 and 7.  Built in to NQC is a function that will randomly choose numbers for you.  For example, 
x = Random(4);         // Generates a random number between 0 and 4
will assign a random number to the variable x, between 0 and 4.  For this program, you should choose a number larger than 0, since you'd like the robot to beep at least once.  For example, if you'd like a number between 1 and 10 stored in variable x, you could use:

x = Random(9)+1;      // Generates a random number between 1 and 10

Use the random number generator to generate a number between 1 and 7 to be sent.  To make it easier to tell if your robot is doing the right thing, we'll use a trick to make the view screen to display the number.  Since the OUT_B port is not being used, set the power level for OUT_B to be the random number you just generated (see the NQC Reference Sheet for how to set the power levels).

Modify the beep.nqc program to display the incoming message as the value for OUT_B, as well as beeping that number of times.

Next, write a program that will turn left if it receives an even number as a message and turns right if it receives an odd number.

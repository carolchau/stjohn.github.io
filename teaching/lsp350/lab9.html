Rotation Sensor

Today's lab focuses on using a new sensor, called the rotation sensor. The rotation sensor measures the amount something has turned.  We'll write several programs to test the rotation sensor and then use it to  go exact distances, independent of how strong the batteries in the robot are.

Design requirements: This lab assumes that the two motors are wired to output ports A and C. Further, each motor is attached to a means of locomotion (legs, treads, feet, etc.).  A rotation sensor should be attached to the shaft of one of your motors and connected to a sensor port.

Testing the Sensor

The rotation sensor allows us to measure how much something has turned.  Like other sensors, you must tell the robot what kind of sensor you are using.  For example, to set up a rotation sensor on an input port you have called ANGLE (that is, you have #define at the top of the program saying which input port is called ANGLE):
   SetSensor(ANGLE, SENSOR_ROTATION);        
The sensor can measure how far something has turned (see p. 28-29 for pictures) in increments of 22.5 degrees.  So, a value for the rotation sensor of 16 occurs when you've turned 16*22.5 = 360 degrees.  So, a full rotation equals a measurement of 16.  If you would like to turn 90 degrees, you would stop when you see a measure measurement of 4.  To test the value of the rotation sensor, we use the same format as for other sensors.  For example, assume ANGLE is the input port for your sensor.

  until (ANGLE == 16);      // Wait until one full rotation has occurred

To start today, write a simple program that moves both motors forward at a very, very slow speed and has a rotation sensor attached to one of the motors.  Use the view button to examine the input from the rotation sensor.  What values does it take?  Change the program (or the motor leads) so that the robot goes backwards.  What happens to the values from the rotation sensor?

Next, write a program that will beep every full revolution of the motors.

Going Precise Distances

With a rotation sensor, you can now write programs where your robot goes precise distances.  This makes going repeatable distances (for example, as in the obstacle course) much easier, since you no longer have to adjust the time to make up for weakening batteries. 

Write a program, using the rotation sensor that will trace out one side of a tile on the floor.  Note that this will likely be several full rotations.  Use your second program from before to get a rough estimate (the number of beeps equals the number of full rotations) and then perfect your program by trial and error.
Next, write a program that will trace out a complete square.

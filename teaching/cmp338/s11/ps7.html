This problem set covers binary search trees. See the class webpage for references.

In this assignment, you will implement useful methods for binary search trees, as well as analyze performance. The necessary files are available on Blackboard.
  1. Rotating Binary Search Trees (60 points)

    Using the base program discussed in class (available on Blackboard), add the following methods:

    • height(N): returns the height of the tree rooted at N
    • rotateLeft(N): rotate the tree to the left at node N, if the right child of N is not null.
    • rotateRight(N): rotate the tree to the right at node N, if the left child of N is not null.

    For a good description of rotating binary trees, see Wiki page.

    Submit your program as a .java file via Blackboard.

  2. Test Cases (20 points)

    Report the following:

    • the height of a tree created by inserting 127 randomly generated numbers (ranging from 1 to 500) are inserted into a binary search tree.
    • the height of the initial tree after 1 left rotations at the root (if possible).
    • the height of the initial tree after 1 right rotations at the root (if possible).
    • the height of the same tree after 3 left rotations at the root (if possible).
    • the height of the initial tree after 3 right rotations at the root (if possible).

    Run the test 10 times and make a table with your average heights as well as the standard deviation for each algorithm. Submit your table via Blackboard.

  3. Analysis (20 points)

    Using your data above, answer the following questions:

    • Do single rotations, on average, shorten the height of the tree? Why or why not?
    • Do three rotations in a row, on average, shorten the height of the tree? Why or why not?
    • The shortest tree possible with 127 keys has height 6, while the tallest possible tree has height 126. How close were your trees to the best (ie shortest) height?

    Submit your answer via Blackboard as a text or word file.

Problem Set 11

CMP 338: Data Structures and Algorithms
Lehman College, City University of New York
Spring 2011

This problem set covers hashing. See the class webpage for references.

  1. Linear Search (10 points)

    Set up a simple program that generates 100 random integers between 1 and 100,000 and stores them in an array. Write a method that takes a integer as input and returns the index if the number occurs in your array, -1 otherwise.

    Write a simple driver program that demonstrates that your program works.

    Submit the program as .java file to Blackboard.

  2. Hash Table Using Chaining (30 points)

    Modify your program above to also include a hash table using chaining. The hash table should be inplemented "by hand" (in constrast to the next section where you implement it using the "built in" classes). You should set your table size to 100 and populate with the same random integers that you place in your array.

    Write a simple driver program that demonstrates that your program works.

    Submit the program as .java file to Blackboard.

  3. HashMap and HashTable (20 points)

    Modify your program above to also include a hash table implemented with HashMap and HashTable. You should populate these with the same random integers that you place in your array and hand-made hash table.

    Write a simple driver program that demonstrates that your program works.

    Submit the program as .java file to Blackboard.

  4. Test Cases (20 points)

    Run your program on the following:

    • the universe is 1,...,10,000; the table/array size (m) is 100; the number of entries is 100; search for the number 1000
    • the universe is 1,...,10,000; the table/array size (m) is 1,000; the number of entries is 1000; search for the number 1000
    • the universe is 1,...,100,000; the table/array size (m) is 1,000; the number of entries is 500; search for the number 1000
    • the universe is 1,...,100,000; the table/array size (m) is 1,000; the number of entries is 100; search for the number 1000
    • the universe is 1,...,100,000; the table/array size (m) is 10,000; the number of entries is 1000; search for the number 1000

    Run the test 10 times and make a table with your average search time for each of the four structures as well as the standard deviation. Submit your table via Blackboard.

    Submit your answers as an excel or text file via Blackboard.

  5. Analysis (20 points)

    Using your data above, answer the following questions:

    • Which structure has the fastest search times?
    • Which structure was easiest to implement?
    • In the future, which would you choose to implement a map class where you store values by their key?
    • What affects running time more, table size or load factor (alpha = m/n)?

    Submit your answers as a text file via Blackboard.

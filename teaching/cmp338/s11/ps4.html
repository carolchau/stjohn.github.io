This problem set covers linked lists. See the class webpage for references.

  1. Linked Lists (30 points)

    Using the singly-linked list done "by hand" from Java Notes as your base, add the following methods:

    • insert a new word at the end of the list
    • print out the first word from the list and remove it from the list
    • add a new word at index i

    Your program should demonstrate that these methods work by printing out the list after each of these operations have been performed.

    To analyze the running time of each method, include the following:

    • code to time each method, in milliseconds, in the main() method (see previous problem sets for examples).
    • to slow down the methods (so we don't have to use such huge data sets for testing), add to each method, Thread.sleep(1) for every time you look at an element of the linked list. This call will cause the program to wait for 1 millisecond before continuing.

    Submit your answer as a .java file via Blackboard.

  2. Test Cases (20 points)

    Calculate how long the following situations take:

    • a list of 10 Strings, adding a new word at index 2
    • a list of 10 Strings, adding a new word at index 5
    • a list of 100 Strings, adding a new word at index 2
    • a list of 100 Strings, adding a new word at index 50
    • a list of 1000 Strings, adding a new word at index 2
    • a list of 1000 Strings, adding a new word at index 500

    Run each test 10 times and make a table with your average as well as the standard deviation for each algorithm. Submit your table via Blackboard.

  3. Analysis (20 points)

    Using the table you created, answer the following questions:

    • Did the size of the list affect the running time for the first two methods? Why or why not?
    • What affected the running time of the third method? If size of the list is fixed, is the running time constant for this method? If the index is fixed, is the running time constant for this method?

    Submit your answer via Blackboard.

  4. Order of Growth (30 points)

    For each of the three methods, analyze their order of growth. That is, calculate a tight upper bound on their growth, written in "big-Oh" notation. Explain and justify your answer.

    Submit your answer via Blackboard.

Problem Set 8

CMP 338: Data Structures and Algorithms
Lehman College, City University of New York
Spring 2011

This problem set covers priority queues and heaps. See the class webpage for references.

In this assignment, you will implement a priority queue for student enrollment at Lehman.
  1. Student Class (20 points)

    At Lehman College, students are assigned registration appointments based on several factors. Make a Student class that holds the following information:

    • name: the name of the student
    • numCredits: the number of credits the student has earned (must be a positive integer)
    • priority: the student's priority for registration (must be a positive integer)
    • athlete: true if the student is on a collge athletics team, false otherwise
    • honors: true if the student is a member of the honors college, false otherwise

    You should also have the following methods:

    • setPriority(): Sets the priority based on the following:
      • numCredits <= 15 (lower freshman), priority is set to 1
      • 15 < numCredits <= 30 (upper freshman), priority is set to 2
      • 30 < numCredits <= 45 (lower sophomore), priority is set to 3
      • 45 < numCredits <= 60 (upper sophomore), priority is set to 4
      • 60 < numCredits <= 75 (lower junior), priority is set to 5
      • 75 < numCredits <= 90 (upper junior), priority is set to 6
      • 90 < numCredits <= 105 (lower senior), priority is set to 7
      • 105 < numCredits <= 120 (upper senior), priority is set to 8
      • 120 < numCredits, priority is set to 9
      If the student is an athelete or an honors student, the student receives an extra 10 points to their priority ranking. For example, a starting freshman who is an athlete, will have priority = 1 + 10 = 11.
    • toString(): returns a string with instance variables. For example, "Student12, 78 credits, athlete, not honors; priority: 16"

    Submit your class as a .java file via Blackboard.

  2. Enrollment Priority System (20 points)

    Write a program to simulate the enrollment priority system. Your program should generate 1000 students at random, and store the students in a priority queue (implemented as an array based heap), using the priority variable as the key. When generating the students, you should:

    • assign the names "Student1", "Student2", etc, so they can be identified later on,
    • randomly generate the number of credits between 0 and 130,
    • randomly assign students to the honor college with a probability of 1% (i.e. we expect 1 out of every 100 students will be a member of the honor college)
    • randomly assign student atheletes with a probability of 5% (i.e. we expect 1 out of every 20 students will be an athelete)

    Use the heapsort algorithm that will print out students, ordered by priority.

    Submit your program as a .java file via Blackboard.

  3. Simulation (20 points)

    The mathematics department has decided to offer a new course called, "Pi: The Intersection of Mathematics and Cooking" that fulfils the mathematics requirement, science requirement, writing requirement, counts as an LEH 300 course, and feeds the student at every class. It is very popular, but there are only 20 seats available in the class.

    Extend your simulation program above. After generating the students and populating the priority queue, remove students from the priority queue until the course fills. You may assume that 10% of the students will sign up for the course. Continue removing students from the queue until the course fills. Report the priority level and number of credits of the last student who enrolls for the course. For example, if student12 from the example above is the 20th student to enroll, your program should report: 

    Class full.  Last student to enroll had priority 16 and 78 credits.

    Submit your program as a .java file via Blackboard.

  4. Test Cases (20 points)

    Run the following tests of your program

    • the original settings: 1% honors, 5% atheletes, 10% demand for Pi course.
    • less atheletes: 1% honors, 2% atheletes, 10% demand for Pi course.
    • more atheletes: 1% honors, 10% atheletes, 10% demand for Pi course.
    • less demand: 1% honors, 2% atheletes, 5% demand for Pi course.
    • more deman: 1% honors, 2% atheletes, 20% demand for Pi course.

    Run the test 10 times and make a table with the average of the credits of the last student to be enrolled (and fraction with honors/athlete status) as well as the standard deviation for each algorithm. Submit your table via Blackboard.

  5. Analysis (20 points)

    Using your data above, answer the following questions:

    • Does the number of athletes in the simulation affect whether a senior can get into the Pi class?
    • Does the number of athletes in the simulation affect whether a sophomore can get into the Pi class?
    • Does the demand for the Pi course in the simulation affect whether a senior can get into the Pi class?
    • Does the demand for the Pi course in the simulation affect whether a sophomore can get into the Pi class?

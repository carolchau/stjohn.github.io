Suggested Syllabus for Analysis of Algorithms Core Course
    

This outline has been prepared for the Computer Science curriculum committee at the Graduate Center, Fall 2000.  This is just a suggested outline and syllabus and does not reflect the current policy of the Computer Science department at the Graduate Center.  For more information about the current course requirements, see the official bulletin at  Graduate Center .
 
 

Prerequisites
    

The prerequisities for this course are the standard courses for an undergraduate degree in computer science.  The ACM has developed a guideline for the  computing curriculum, listing those courses.  Students are expected to have had basic algorithm analysis, algorithmic strategies, fundamental computing algorithms, and fluent in an object-oriented programming language.  In particular, students should know (list taken from the  ACM Computing Curriculum 2001):

AL1. Basic algorithmic analysis (core -- 4 hours)

  • Asymptotic analysis of upper and average complexity bounds
  • Identifying differences among best, average, and worst case behaviors
  • Big "O" and little "o" notation
  • Standard complexity classes
  • Time and space tradeoffs in algorithms
  • Using recurrence relations to analyze recursive algorithms
AL2. Algorithmic strategies (core -- 6 hours)
  • Brute-force algorithms
  • Greedy algorithms
  • Divide-and-conquer
  • Backtracking
  • Branch-and-bound
  • Heuristics
AL3. Fundamental computing algorithms (core -- 12 hours)
  • Simple numerical algorithms
  • Sequential and binary search algorithms
  • Quadratic sorting algorithms (selection, insertion)
  • O(N log N) sorting algorithms (Quicksort, heapsort, mergesort)
  • Hash tables, including collision-avoidance strategies
  • Binary search trees
  • Representations of graphs (adjacency list, adjacency matrix)
  • Depth- and breadth-first traversals
  • Shortest-path algorithms (Dijkstra's and Floyd's algorithms)
  • Transitive closure (Floyd's algorithm)
  • Minimum spanning tree (Prim's and Kruskal's algorithms)
  • Topological sort
  • Pattern matching and string/text algorithms
  • Numerical approximation algorithm

NYU's UG Course:
V22.0455 ANALYSIS OF ALGORITHMS

 Prerequisite: V22.0310. 4 points. Principles and techniques of worst-case analysis and average behavior

161. Design and Analysis of Algorithms-Efficient algorithms for sorting, searching, and selection. Algorithm
analysis: worst and average case analysis. Recurrences and asymptotics. Data structures: balanced trees, heaps,
etc. Algorithm design techniques: divide-and-conquer, dynamic programming, greedy algorithms, amortized
analysis. Algorithms for fundamental graph problems, e.g., depth-first search, connected components,
topological sort, shortest paths. Possible topics: network flow, string searching, parallel computation.
Prerequisite: 103 or 109, Statistics 116.

NYU's Course:
G22.1170 Fundamental Algorithms
 Prerequisites: minimum of one year's experience with a high level language such as Pascal or C,
 familiarity with recursive programming methods and with data-structures (arrays, pointers, stacks, queues,
 linked lists, binary trees).

 Reviews a number of important algorithms with emphasis on correctness and efficiency Sorting algorithms:
 mergesort, binsort, quicksort, heapsort. Binary search, hash-coding. Binary search-trees and
 balanced-tree strategies. Binary tree traversal: infix and postfix notation. Dynamic sets: Union-find data
 structures. Graphs: connectivity, spanning trees, shortest paths, transitive closure. Programming practice.

G22.3520 Honors Analysis of Algorithms
 Prerequisite: an undergraduate algorithms course, or G22.1170, or a strong mathematical background

 The course reviews and elaborates on the design of algorithms and their data structures. The review
 material covers searching and sorting, and the more fundamental graph algorithms (covered in G22.1170
 and G22.1171). Techniques for analyzing the complexity of algorithms are covered in depth, and include
 advanced topics on recurrence equations and NP-complete problems. Additional material includes
 advanced topics on lower bounds, randomized algorithms, amortized algorithms, and data structure
 design. Specific applications include algorithms for union-find, pattern matching, polynomial arithmetic,
 network flow, and matching.

Donuts and Cookies

The book develops the iRock and donut example in the first three chapters. We will use the files from Chapter 2 and 3:

http://headfirstlabs.com/books/hfjs/ch03.php

Using the donut example as a base, modify the JavaScript to remember the user's name when they return. The simplest way to do this is to mimic what was done for iRock. Recall that the last version of the iRock asks the user their name and stores it in cookie.

The steps that need to be done here:
  • When the name changes, store it in a cookie (how can you tell when a field changes?)
  • When the page loads, if the name is known, put it in the input field on the form (Hint: Look at the iRock example for checking if the cookie exists. You will want to make this change when the onload event has been triggered-- use the <body> element.)

Finding Bugs

An important part of programming is to be able to locate and fix bugs. The goal of today's lab is to find (and fix!) all the bugs in the following file:

donutsBugs.html

This page should prompt a user for their name and the donuts and bagels they wish to purchase. Currently, there are too many bugs in the file for it to do this. Using the error console in the Mozilla Firefox browser, find each bug and fix it.

Some useful things to remember when hunting for bugs:

  • If you would like the graphics to appear and the style to look as before, save this file to the directory containing your donut page from before.
  • Bugs often occur the line above which the error console reports. So, look both at the line reported and the line above.
  • A single bug can cause multiple error messages. So, fixing the first one reported on the error console will often get rid of multiple error messages.
  • Use the "clear" button on the error console between multiple runs of your program. This will keep you from confusing old errors with the current ones.

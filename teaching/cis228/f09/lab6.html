Today's lab wraps up what we've learned this far and discusses proper form for HTML. To verify that pages are in good form, we will use an HTML validator on several different pages, correcting problems as we go.

Using Standard HTML

Before starting the lounge page exercises, you will need to download the files provided on the book's webpage to a floppy drive or USB keychain. If you haven't already done so, the book's website is:

http://headfirstlabs.com/books/hfhtml/index.php
We're going to work with the index.html file in Chapter 6 (the lounge example, revisited).

  1. Add in the <DOCTYPE> tag (in doctype.txt or book) at the top of the lounge.html file.
  2. Use the validator, http://validator.w3.org, to check if the page meets standards.
  3. Add in the suggested <META> tag, and try the validator again.
  4. Fix any mistakes (hint: inline elements must be in blocks) and try the validator again.

All-About-Me Website

The final activity for today's lab we will to choose a page that you've written for a previous lab and modify it to meet HTML 4.01 standards. Use the validator to verify that the page is in standard HTML.

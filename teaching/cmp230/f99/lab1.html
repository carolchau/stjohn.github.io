General Information About the Lab

The machines in the lab run a modified version of the Windows operating system. Many features of Windows have been disabled for security reasons. For example, you cannot save anything to the hard drive and you cannot "right click."

The department has the following policies for the lab:

  • There is no eating or drinking in the lab.
  • Due to restricted class sizes, children are not allowed in the computer labs.
  • Do not waste paper-- only print files that directly relate to the class.
  • Do not enter the lab until the scheduled class time. If the lab is open before class time, it is because another class is using it or for maintenance.
  • Always log out and turn off the monitor when you leave the lab.

Getting Started

First, turn on the computer and monitor-- buttons for both are in the front. It will take a few moments for the computer to get started.

When the computer is ready, you will see a login screen. Type in c230 for the username and leave the password blank. Next, you will see a message to insert a disk into drive A:. If you do not have disks with you, there are some at the front of the classroom you may borrow. They must be returned at the end of the lab.

Finally, you should see a Windows desktop. Click on the icon marked A: to access the floppy disk. Click on the icon marked Student Data. This includes files used for classes taught in the lab. Choose the c230 folder. You will see lots of JBuilder files. Find the one called SavitchIn and drag it to the A:. This will copy the classes used by the book onto your disk. We will use these classes a lot, so, you will want a copy of your own in the same folder that contains your java programs.

Using JBuilder

Click on the JBuilder icon. This starts the Java programming environment we are using this semester. It's a sophisticated package with many features we will not use at first. We will start with just creating a simple java file. To create a file choose Open/Create from the File menu. Give your file the name FirstProgram.java.

Type at the very top of the file: 

    /* <your name here>
   <your email here>
   CMP 230-1, 1 September 1999
*/

import SavitchIn.*;
This is a comment saying whose program it is, followed by a line which tells JBuilder that you would like to use the classes that come with the textbook. Now, type in the program on p 18 of the book-- it's a simple program that is the basis of some of the homework for next week.

Save your program (Save under the File menu).

Before our program will compile, we need to make sure that the book's package SavitchIn is open and compiled. To open SavitchIn, choose Open/Create from the File menu, and choose SavitchIn from the A: drive (you may have to change the file type to "All Files" at the bottom of the Open/Create dialog box). You can look at the file by clicking on its name in the left hand box.

When large projects are assembled, it's referred to as building (it you recompile everything) or making (if you only recompile the parts that have been updated. To compile SavitchIn, choose Build "SavitchIn" from Build menu. If it compiles successfully, there will be a message about halfway down the screen saying so. Next, we want to compile the program. So, choose Build "FirstProgram" from Build menu. If your program compiles successfully, there will be a message about halfway down the screen saying so. If it didn't, there will be an error message in the bottom right hand corner of the screen. You will have to fix all errors before going onto the next step (ask for help if this happens).

Once your program compiles successfully, you can run it by choosing Run "FirstProgram" from the Run menu.

E-mail in the Lab

For security reasons, the graphical e-mail programs have been disabled. To mail programs, you can do it from home or another lab (just attach them as a text file in Netscape Mail, Eudora, or your favorite mail program), or use a terminal window in the lab.

To do this from the lab, first Copy your program in JBuilder (Copy is under the Edit menu). Then, click on the Terminal icon. Choose the Open option under the File menu and type the name of your host (e.g. alpha.lehman.cuny.edu). Next, type mail stjohn. Using Paste under the Edit menu, paste the program into the mail message. Type a carriage return, a period, a carriage return to end the message. To log out of the terminal, type logout.

Logging Out

Before leaving the lab, you need to log out and shut down your computer. First, you should quit all applications. Then, go to the START menu in the bottom left hand corner and select Shut Down. Also, turn off the monitor (button is on the front at the bottom right).

Remember to pick up any files you sent to the printer and to take any personal belongings. It is difficult to get back into the lab later (since it's locked when not in use), so, it's worth making sure you haven't forgotten anything.
